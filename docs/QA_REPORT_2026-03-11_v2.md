### 2. `GET /api/auth/me` returns 404 on deployed Render backend
The local codebase has the route (`backend/routes/auth.js:274`) properly mounted (`server.js:87`). However, the live Render backend at `https://tradvue-api.onrender.com/api/auth/me` consistently returns:
```json
{"error":"Route not found"}  →  HTTP 404
```
Even with a Bearer token, it returns 404 instead of 401/403 (which `requireAuth` should produce). This indicates the deployed backend is either an older build or the auth routes are failing to load.  
- `backend/routes/auth.js:273–300` — route exists  
- `backend/server.js:87` — mount exists  
**Fix:** Redeploy backend to Render; check Render build logs for errors loading `routes/auth.js`.

- `Tooltip.tsx:33` — `zIndex: 9999`
- `CookieConsent.tsx:48` — `zIndex: 9999`
- `PageLoader.tsx:67` — `zIndex: 9999`
