# FREE Financial News Sources & APIs for TradVue

Research compiled March 2026 for free-tier financial news APIs and sources suitable for a trading platform.

---

## 🏆 Top Tier (Recommended)

### 1. **NewsAPI (Financial Mode)**
- **URL:** https://newsapi.org
- **Free Tier:** 100 requests/day, articles from 38 news sources
- **Data Provided:** News articles with headlines, descriptions, images, sources
- **Real-time:** Near real-time (15-30 sec delay)
- **Integration Difficulty:** Easy
- **Legal:** Fully legal, attribution required, no commercial redistribution
- **Notes:** Great general coverage; finance category includes market news. Filters available.
- **Ranking Score:** 8/10 for trading

### 2. **Finnhub** (Already Using)
- **URL:** https://finnhub.io
- **Free Tier:** 60 API calls/minute
- **Data Provided:** Market data, company news, earnings, IPO calendar, economic calendar
- **Real-time:** Real-time
- **Integration Difficulty:** Easy
- **Legal:** Fully legal, must provide API attribution
- **Notes:** Excellent for stocks and forex; already integrated
- **Ranking Score:** 9/10 for trading

### 3. **AlphaVantage**

## 🎯 Recommended Integration Stack

**For Best Free Coverage:**

1. **Primary:** Finnhub (already integrated) + NewsAPI
   - Covers stocks, market news, company events
   - Easy to implement
   - Real-time data

2. **Secondary (Crypto):** CoinGecko + CryptoCompare
   - Complete crypto coverage
   - Real-time prices and sentiment
   - Free tier is generous

3. **Tertiary (Economics):** FRED API
   - Macro-level indicators
   - Long-term trend analysis
   - No rate limits on free tier

4. **Optional (RSS):** Seeking Alpha + Yahoo Finance RSS
   - Aggregates news articles
   - Backup to API failures
   - Easy to parse and display

## 🚫 NOT Recommended

- **Investing.com:** Scraping violates ToS (legal risk)
- **StockTwits API:** Too restrictive, limited access
- **Twitter API for sentiment:** Strict commercial restrictions
- **Any paid-first services:** Focus on truly free tiers

## 💡 Implementation Notes

### Rate Limiting Strategy
- Implement request queuing to respect rate limits
- Cache data for 30-60 seconds to reduce API calls
- Use multiple sources as fallbacks

### Legal Compliance
- Always provide source attribution in UI
- Follow each API's ToS regarding commercial use
- Display disclaimers about data delays
- Consider Terms of Service updates quarterly

### Performance Tips
- Batch requests when possible
- Use WebSockets for real-time updates (if supported)
- Store historical data locally to reduce API calls
- Implement circuit breakers for failed sources
